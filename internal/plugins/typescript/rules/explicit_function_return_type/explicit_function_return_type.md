# explicit-function-return-type

## Rule Details

Require explicit return type annotations on functions and class methods.

## Options

- `allowExpressions` (default: `false`)
- `allowTypedFunctionExpressions` (default: `true`)
- `allowHigherOrderFunctions` (default: `true`)
- `allowIIFEs` (default: `false`)
- `allowDirectConstAssertionInArrowFunctions` (default: `true`)
- `allowConciseArrowFunctionExpressionsStartingWithVoid` (default: `false`)
- `allowFunctionsWithoutTypeParameters` (default: `false`)
- `allowedNames` (default: `[]`)
